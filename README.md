# Window Runner

乗り物の窓の外を、キャラが走り抜ける — **タップだけで遊べる横スクロールアクションゲーム**の
プロトタイプです（Linear: SOT-1519）。

幼少期の「乗り物の窓の外でキャラが走る想像遊び」をゲームにしたもの。障害物を避けるのではなく、
**地形の切れ目に落ちないように渡って**進みます。

## 遊び方

| 操作 | アクション |
| --- | --- |
| タップ | ジャンプ |
| 連続タップ | 二段ジャンプ（Lv.2 で解放） |
| 長押し | パラシュート滑空（Lv.3 で解放） |
| （地上でタップ長押し） | 高速ダッシュ（Lv.4 で解放） |

- キャラは自動で前進します。地形の切れ目をジャンプで渡り、ステージ終点まで走り切るとクリア。
- ステージクリアで経験値を獲得し、レベルアップすると新しいアクションが解放されます。
- ステージごとに「乗り物」が変わり、背景の速度・雰囲気が変化します
  （徒歩 → 車 → 電車 → 新幹線 → 飛行機 → 宇宙）。宇宙は低重力でジャンプが伸びます。

## 実行

静的な Web アプリです。ビルド不要。ローカルサーバーで開いてください（ES モジュール読み込みのため
`file://` ではなくサーバー経由が必要です）。

```bash
npm start        # http://localhost:8080 を開く
# もしくは
python3 -m http.server 8080
```

## 動作環境

- ランタイム: Node.js（`npm test` の Node 標準テストランナー用）と Python 3（`npm start` の
  `python3 -m http.server 8080` 用）が必要です。
- モジュール形式: `package.json` は `"type": "module"` を指定しており、ブラウザ側の JavaScript は
  ES モジュールとして読み込まれます。
- エントリポイント: ブラウザで開く入口は `index.html`、ゲームの描画・入力・ループ処理は
  `src/game.js` から開始します。
- 実行方法: `npm start` でローカルサーバーを起動し、`http://localhost:8080` をブラウザで開きます。

## テスト

ゲームのコアロジック（ステージ構成・レベル/経験値・地形生成・当たり判定・物理）は
DOM に依存しない純粋関数として `src/core.js` に分離しており、Node 標準テストランナーで検証します。

```bash
npm test
```

## 構成

```
index.html        エントリ（Canvas + UI）
src/core.js       純粋ロジック（ステージ / レベル / 地形 / 物理 / 判定）— テスト対象
src/game.js       描画・ゲームループ・入力（core.js を利用）
test/core.test.js Node 標準テスト
```

## 完成定義（DoD）との対応

- [x] タップでジャンプできる
- [x] 地形の切れ目を渡って進める
- [x] ステージクリアでレベルアップする
- [x] レベルに応じてアクションが増える（ジャンプ / 二段ジャンプ / 滑空 / ダッシュ / 壁キック）
- [x] 乗り物の種類に応じて背景が変わる
- [x] 最低3ステージが遊べる（6ステージ実装）

## 技術メモ

要件では iOS（Swift / SpriteKit または Unity）が挙げられていますが「相談可」とされており、
この開発環境（Linux）では iOS ネイティブのビルド・動作検証ができません。そのため、DoD をすべて
満たしつつブラウザで実行・検証できる **HTML5 Canvas + JavaScript** の静的 Web プロトタイプとして
実装しています。ネイティブ iOS（SpriteKit/Unity）版が必要な場合は、Mac / 実機環境で別途対応します。
